  <action>
Create `src/lib/services/llm/ollama.ts` with `OllamaProvider` class implementing `LLMProvider`:

**chat() method:**
- Use browser `fetch('http://localhost:11434/api/chat', ...)` with streaming enabled
- Request body: `{ model: options.model || 'qwen2:0.5b', messages, stream: true, temperature: options.temperature ?? 0.7, top_p: options.topP ?? 0.9 }`
- Response handling: `ReadableStream.getReader()` with `TextDecoder`
- JSON-per-line parsing: split buffer by `\n`, parse each complete line, **keep incomplete lines in buffer**
- Yield `json.response` chunks until `json.done === true`
- Use `async *` generator syntax (not callbacks)

**models() method:**
- Fetch `http://localhost:11434/api/tags`
- Map response.models to `Model[]`: `{ id: m.name, name: m.name, contextWindow: 32000, size: m.size }`

**tokenCount() method:**
- POST to `http://localhost:11434/api/tokenize` with `{ model: 'qwen2:0.5b', prompt: text }`
- Return `response.tokens.length`

**isAvailable() method:**
- Try fetch with `AbortSignal.timeout(2000)`
- Return `true` if 200 OK, `false` if error/timeout

Use research file patterns as reference. Handle errors with descriptive messages (connection refused, model not found, etc.).
