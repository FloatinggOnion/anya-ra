**Changes to NotesEditor:**

1. **Add props:**
   ```typescript
   interface Props {
     type: 'document' | 'paper'  // NEW: what are we editing?
     id: string                  // NEW: docId or paperId
     content: string
     onChange: (content: string) => void
   }
   ```

2. **Update auto-save logic:**
   - In `onChange` handler, determine which store function to call:
     ```typescript
     async function handleChange(newContent: string) {
       onChange(newContent)
       
       // Route to correct store based on type
       if (type === 'document') {
         // Call saveDocument from documents store
         await saveDocument($workspace.path, id, newContent)
       } else if (type === 'paper') {
         // Call saveNote from notes store (existing)
         await saveNote($workspace.path, id, newContent)
       }
     }
     ```

3. **Add visual context indicator:**
   - Display in editor header: "Editing: {type} — {id or title}"
   - For documents: show title from documents store
   - For papers: show paper title from papers store
   - Example: "Editing document: My Literature Review" or "Editing notes for: Attention Is All You Need"

4. **Update Cmd+S behavior:**
   - Keep existing Cmd+S handler (manual save trigger)
   - Still routes to correct save function based on type

5. **Import new store function:**
   ```typescript
   import { saveDocument } from '../../stores/documents'
   ```

6. **Backward compatibility:**
   - Existing code that passes type='paper' continues to work
   - Default (if type not specified) → treat as 'paper' (current behavior)

**Implementation notes:**
- The debounce happens in the store functions (saveDocument, saveNote), not in the component
- Component just calls onChange + the appropriate save function
- No changes to CodeMirror editor itself (still same theme, extensions, etc.)
- Visual indicator is optional; core functionality is context-aware routing

**What NOT to change:**
- CodeMirror configuration
- Keyboard shortcuts (except they route to right store)
- Editor appearance or styling
- Extension setup (markdown highlighting, etc.)

**Decision reference:** Per D-05 (editor reuse, context-aware switching).
