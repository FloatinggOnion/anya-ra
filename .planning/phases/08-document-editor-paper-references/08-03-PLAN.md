  <name>Task 9: Create citation validation service with fuzzy matching</name>
  <files>src/lib/services/document-validation.ts</files>
  <action>
Create `src/lib/services/document-validation.ts` that parses citations, validates against papers store, and generates decorations.

**Functions to export:**

1. **parseCitations(content: string): Citation[]**
   - Input: Document markdown content (string)
   - Uses regex: `\[cite:\s*([^\]]+)\]` to find all citation patterns
   - Returns: Array of {citationText, position, content} where position is char offset
   - Example: "As noted [cite: Attention Is All You Need], transformers..." 
     → [{citationText: "Attention Is All You Need", position: 9, content: "[cite: Attention Is All You Need]"}]

2. **validateCitations(citations: Citation[], papers: Paper[]): LinkMetadata[]**
   - Input: Parsed citations + current papers store
   - For each citation, fuzzy match against paper titles to find paperId
   - Returns: LinkMetadata[] with {citationText, paperId, position, status}
   - Fuzzy match algorithm: substring match (case-insensitive) or edit distance < 2
   - If no match: paperId = null, status = 'missing'
   - If match: paperId = paper.id, status = 'valid'

3. **getFuzzySuggestions(citationText: string, papers: Paper[]): Paper[]**
   - Input: Citation text + papers store
   - Returns: Array of top 3 Paper suggestions based on similarity
   - Used for hover tooltip: "Did you mean: [Paper 1], [Paper 2], [Paper 3]?"
   - Return only if paperId is null (broken reference)

4. **createDecorations(links: LinkMetadata[]): Decoration[]** (optional, could be in CodeMirrorDecorations.ts)
   - Input: LinkMetadata array
   - Returns: CodeMirror Decoration objects for red underline on invalid citations
   - Example: For each link with status='missing', create red wavy underline decoration

**Implementation details:**

- Citation interface:
  ```typescript
  interface Citation {
    citationText: string
    position: number      // Character offset in content
    content: string       // Full "[cite: ...]" text
  }
  ```

- Fuzzy match algorithm: Start simple (substring match), can optimize later
  - Exact match (case-insensitive) → paperId
  - Substring match → paperId
  - Edit distance (Levenshtein) < 2 → paperId (if library available)
  - No match → null

- Validation happens on every keystroke (debounced by caller in DocumentEditor)

**Error handling:**
- parseCitations: No validation, just extraction (malformed citations are OK, will fail validation)
- validateCitations: Graceful degradation (paper not found → status='missing')
- getFuzzySuggestions: Return empty array if no matches

**Testing considerations (for Wave 3 verify):**
- Input: "[cite: Attention Is All You Need]" with papers store containing exact match → status='valid'
- Input: "[cite: Attention]" with papers store containing "Attention Is All You Need" → status='valid' (substring)
- Input: "[cite: Unknown Paper]" with no papers store match → status='missing'
- Input: "No citations here" → empty array

**Decision references:** Per D-01 (citation syntax), D-03 (real-time), D-04 (storage).

**New functions to add:**

   - Calls saveDocument() to write content to .md file
   - Creates DocumentSidecar with links array
   - Calls documents-io.saveDocumentSidecar() to write .links.json
   - Debounce applies to both saves (they happen together)
   - Updates store with latest links

2. **loadDocumentWithLinks(workspacePath, docId): Promise<{document, links}>**
   - Calls loadDocument() + loadDocumentSidecar() in parallel
   - Returns both document content and links metadata
   - Used when opening a document to restore validation state

3. **updateLinks(docId, links): void**
   - Updates only the links array in-memory (for validation state)
   - Does NOT persist to disk immediately (validation is ephemeral)
   - Allows validation to run independently without I/O

**Modified functions:**

4. **saveDocument() — no changes to signature**
   - Existing behavior unchanged (backward compatible)
   - Just saves content, not links
   - Used by default auto-save

**Implementation notes:**

- saveDocumentWithLinks() wraps both .md and .links.json saves
- Links array is never null (initialized as empty [] on new document)
- Debounce applies to both files together (atomic save)
- If .links.json write fails, content is already saved (eventual consistency is OK for Phase 8)

**Error handling:**
- If sidecar save fails: log error, content is safe, links will be re-validated on next open
- If content save fails: throw (don't save sidecar without content)
