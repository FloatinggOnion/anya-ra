# Phase 5: Knowledge Graph - Plan Summary

Phase 5 adds a visual, persistent knowledge graph canvas to Anya-RA using @xyflow/svelte 1.5.1 (the only new dependency - natively Svelte 5 runes-compatible). The implementation follows a 5-wave build order: Wave 1 installs the library and creates TypeScript discriminated-union types (GraphNode, GraphEdge, GraphFile) while simultaneously adding two Rust commands (save_graph_file / load_graph_file) that write a single {workspace}/graph.json; Wave 2 builds a Svelte writable store with debounced auto-save (persistGraph) plus orphan-filtering load (initializeGraph) plus a graphFitOnInit boolean store that controls whether SvelteFlow calls fitView on mount (true on first run, false when a saved graph file is loaded so initialViewport restores the saved position), and creates the three custom node Svelte components (PaperNode, ConceptNode, NoteNode) plus a TypedEdge with relation-colored labels, all wired through a stable nodeTypes.ts module-scope file to prevent the remount-flicker pitfall; Wave 3 assembles GraphCanvas.svelte with the controlled $state.raw pattern (onnodeschange/onedgeschange/applyNodeChanges), duplicate-edge guard, keyboard delete, MiniMap with per-type colors, a NodeEditor modal, and a required FlowController.svelte child component - useSvelteFlow() uses Svelte getContext() which is only valid inside the SvelteFlow tree, so FlowController.svelte is rendered as a child of SvelteFlow, calls useSvelteFlow(), and passes the flow instance up to GraphCanvas via an onready callback, with _persist() and handleFitView() gated on flowInstance != null; Wave 4 integrates everything - adding a Graph fourth tab to MainPanel.svelte, an Add to Graph button (with In Graph feedback) on PaperDetail.svelte, and wiring initializeGraph() into App.svelte onMount after papers load; Wave 5 applies polish (Delete key alias, empty-state hint panel, hover affordance on PaperNode, MiniMap node colors). Key architectural decisions: $state.raw() for SvelteFlow arrays (required by the library to avoid infinite re-render loops), nodeTypes at module scope in a separate .ts file (prevents remount flicker), FlowController.svelte child component for safe useSvelteFlow() access (Svelte context is parent-to-child only), graphFitOnInit store to resolve the fitView/initialViewport conflict (only one may win - fitView runs after initialViewport and overrides it), denormalized paper fields in node data with papers-store as authority on load (orphan safety), and debounced 300ms save triggered from the SvelteFlow controlled handlers (not from individual store subscriptions).
